Creating a grid

To set up a grid, you need to have both, a grid container and grid items

The grid container will be a parent element that contains grid items as children

To turn a HTML element into a grid container, you have to set the element's display property to one of the two values. grid - for a block-level grid. inline-grid - for an inline grid

Creating columns

By default, grids contain only one column. If you were to start adding items, each item would be put on a new row. To add columns, you need to explicitly define teh number of rows, columns in the grid

You can define the columns for the grid by using the CSS property, grid-template-columns.

Creating Rows

To specify the number and size of rows, we are going to use the property grid-template-rows

When using percentages in the specifying of rows/columns properties, remember that rows are defined as a percentages of grid's height, columns are defined as a percentage of grid's width

Grid Template

This is a shorthand property. Grid-template can replace the previous two CSS properties.

When using grid-template, the values before the slash determines the size of each row. The values after determines the size of each column.

Fraction

As you have seen a few types of responsive units, such as %, em and rem. CSS Grid introduces a new relative sizing unit - fr (fraction)

Using fr unit, we can define the size of columns, and rows, as a fraction of the grid's length and width. This unit was specifically created for use in CSS grid

Using fr makes it easier to prevent grid items from overflowing the boundaries of the grid

250px height - 80px width
250px height - 240px width
250px height - 80px width
125px height - 80px width
125px height - 240px width

The example above, has 3 rows and 3 columns.

The rows are splitting up the available 400 pixels of height into 4 parts. First row gets 2 of those part, second and third gets one each.

Each column's width is a fraction of the available space

Just think, of the space is being split into parts (the fr)

You can mix and match the units together, a set of columns can be described in fr, %, px

Repeat

repeat() is a function. It can duplicated the specifications for rows or columns, a given number of times

Instead of writing 100px 100px 100px (which specifies three columns to be each 100px wide), you can do repeat(3, 100px)

You can include a second parameter too, so repeat(2, 20px, 50px)

This will create 4 columns, where the first and third is 20px wide, second and fourth is 50px wide

You can use it with fr too

Minmax

Sometimes you might want a grid to resize based on the size of your web browser

Situations such as preventing a row or column from getting to big or small.

100px minmax(100px, 500px) 100px

The value above tells us that the 1st and 3rd columns will always be 100px wide, no matter the size of the gird. But the second column will vary in size as the overall grid resizes. The second column will always be between 100 and 500 px wide.

Grid Gap

So far in our grids, there hasn't been any space between the items in our grid. Row-gap or column-gap will put blank space between every row or column in the grid

There is also a shorthand property, gap. It doesn't have the slash like other shorthand properties. The first specifies the row gap, then the column gap. If only one value is given, it will set both the column and row gap to that value

Grid Items

so far the examples we have used, each item in the grid has taken up one square. This does not always need to be the case. We can drastically change the look of the grid by making the grid item take up more than one row or one column.

Multiple Row Items

Grid-row-start or grid-row-end, we can make a single grid item take up multiple rows. WE ARE NOT APPLYING TO THE GIRD CONTAINER BUT RATHER, ADDING CSS TO THE ITEM IN THE GRID

For example. If a grid has 5 rows, the grid row lines range from 1 to 6.

The value of grid-row-start should be the row at which you want the grid item to begin. The value for grid-row-end should be one greater than the row at which you want the grid item to end. Therefore, an element that covers rows 2,3,4 should have declarations such as grid-row-start: 2, grid-row-end: 5

It is possible for the value of grid-row-start to be greater than grid-row-end, it will still cover, but look awkward on the declarations. they can also have negative values, check the MDN docs

Grid Row

We can use grid-row as a shorthand for grid-row-start and grid-row-end

The starting row goes before the / and the ending row goes after it.

It shall include the gap measurements too if it exists. An item spanning two rows of height 100px each with a 10px gap, the item will be a total height of 210 px

Grid Column

Same for grid-column

when using these properties, you can use keyword span to start or end a column or row

grid-column: 4 / span 2. this code tells us the item begins in column 4,takes up 2 columns of space. thus item occupies columns 4 and 5

Span helps us avoid errors when determining the ending of the grid line. If you know where you wan your grid item to start and how long it should be, use span.

Grid Area

This is a shorthand for all of the grid-row/column-start/end property

Grid area takes four values. First value, grid-row-start, second value, grid-colum-start, third value, grid-row-end, fourth value, grid-column-end

grid-area: 2 / 3 / 4 / span 5

The item will start row 2 and end row 4. row 4 is not included though, only 2 and 3. the item also start on column 3 spanning 5 columns, so 3,4,5,6,7.

Justify Items

There are two axes in the grid layout. The column or block axis. The row or inline axis.

The column axis stretches from top to bottom across the web page

The row axis stretches from left to right across the web page

Justify items positions grid items along the inline or row axis.

This means it positions items from left to right across the web page

This property is declared on grid containers

  • Start. Aligns grid items to the left side of the grid area
  • End. Aligns grid items to the right side of the grid area
  • Center. Aligns grid items to the center of the grid area
  • Stretch. Stretches all the grid items to fill the grid area
  • There are more values and it can be found on MDN. These are the main ones listed above

Justify Content

In this exercise we will learn how to position a grid within its parent element

We can use justify-content to position the entire grid along the row axis. This property is declared on grid containers

The values it accepts are:

  • Start. Aligns grid to the left side of the grid container
  • End. Aligns grid to the right side of the grid container
  • Center. Centers the grid horizontally in the grid container
  • Stretch. Stretches the grid items to increase the size of the grid to expand horizontally across the container
  • Space-around. Includes an equal amount of spae on each side of a grid element
  • Space-between. Includes an equal amount of space between grid items and no space at either end
  • Space-evenly. Places an even amount of space between grid items and at either end

Align Items

Align items shows us how to position grid items from top to bottom

Align-items is a property that positions grid items along the block/column axis. It is declared on grid containers

The values it accepts are:

  • Start. Aligns grid items to the top side of the grid area
  • End. Aligns grid items to the bottom side of the grid area
  • Center. Aligns grid items to the center of the grid area
  • Stretch. Stretches all items to fill the grid area

Align Content

Same as Justify Content, but top to bottom.

Justify Self and Align Self

The justify-items and align-items properties specify how all grid items contained within a single container will position themselves along the row and column axes.

Justify-self specifies how an individual element should position itself with respect to the row axis. This property overrides justify-items for any item on which it is declared. Align self the same but respect to the column axis.

These properties are declared on grid items.

  • Start. Positions grid items on the left side/top of the grid area
  • End. Positions grid items on the right side/bottom of the grid area
  • Center. Positions grid items on the center of the grid area
  • Stretch. Positions grid items to fill the grid area

Implicit vs Explicit Grid

So far we have been explicitly defining the dimensions and quantities of our grid elements using various properties

There will be instances where we don't know how much information we are going to display.

this is where implicit grid comes in handy. Implicit grid is an algorithm built into the specification for CSS Grid, that determines default behavior for the placement of elements

The default behavior of the implicit grid is, items fill up the rows first, adding new rows as necessary. New rows will be tall enough to contain the content within them.

Grid Auto Rows, Grid Auto Columns

CSS Grid provides two properties to specify the size of grid tracks added implicitly. Grid-auto-rows, grid-auto-columns. These properties are declared on grid containers

Grid-auto-rows specifies the height of implicitly added grid rows.

Grid-column-rows specifies the width of implicitly added grid columns.

The values accepted are, px, %, fr, repeat()

Grid Auto Flow

In addition to setting the dimensions of implicitly-added rows and columns, we can specify the order in which they are rendered

Grid-auto-flow specifies whether new elements should be added to rows or columns, and it is declared on containers

The values that are accepted are:

  • Row - specifies the new elements should fill rows from left to right and create new rows when there are too many elements (default).
  • Column - specifies the new elements should fill columns from top to bottom, and create new columns when there are too many elements
  • Dense - this keywords invokes an algorithm that attempts to fill holes earlier in the grid layout if smaller elements are added.
