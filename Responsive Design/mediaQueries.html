Responsive Web Design

When websites respond to the size of the screen, it is called responsive website.

Viewport Meta Tag

Viewport is the total viewable area for a user, this area varies depending on device. The viewport is smaller on mobiles devices and larger on a desktop.

Based on the size of the viewport, the meta tag is used to instruct the browser on how to render the webpage's scale and dimensions.

Imagine a webpage is 960 px and the device is 320px wide, adding a viewport meta tag, will squish the content down until it is 320 px. There is no need for the user to zoom out and view the whole page.

Meta tag goes inside the head element

Let us break down the meta tag. Here is the example: meta name="viewport" content="width=device-width, initial-scale=1"

  • name="viewport" attribute tells the browser to display the web page art eh same width as its screen
  • The content attribute defines teh values for the meta tag, including width and initial-scale
  • The width=device-width key-value pair controls the size of the viewport in which it sets the width of the viewport to equal the width of the device
  • The initial-scale=1 key-value pair: sets the initial zoom level

Therefore the viewport meta tag allows our web pages to be responsive and adapt a site's contents to the users' screen size.

Media Queries

CSS uses media queries to adapt a website's content to different screen sizes.

With media queries, CSS can detect the size of the current screen and apply different CSS styles depending on the width of the screen.

@media only screen and (max-width: 480px) { body { font-size: 12px; } }

The example above defines a rule for screens smaller than 480 px (approx the width of many smartphones in landscape orientation)

Lets break down the example down

  • @media is a keyword. It begins a media query rule and instructs the CSS compiler on how to parse the rest of the rule.
  • only screen. This indicates what types of devices should use this rule. There used to be other types but now, screen is the media tpe always used for displaying content, no matter the type of device. The only keyword is added to indicate that this rule only applies to one media type (screen).
  • and (max-width : 480px). This part of the rule is called a media feature. It instructs the CSS compiler to apply the CSS styles fo devices with a width of 480px or smaller. Media features are the conditions that must be met in order to render the CSS within a media query.
  • CSS rules are nested inside the media query curly braces. The rules will be applied when the media query is met. So, in the example above, the text in the body element is set to a font-size of 12px, when the user's screen is LESS than 480px

Range

Specific screen sizes can be targeted by setting multiple width and height media features. By using multiple widths and heights, a range can be set for a media query.

@media only screen and (min-width: 320px) and (max-width: 480px) { /* ruleset for 320px - 480px */ }

The example above applied the css rules onl when the screen size is between 320 pixels and 480 pixels. The use of and allows us to chain two requirements together.

The example above can also be written like this:
@media only screen and (min-width: 320px) { /* ruleset for >= 320px */ } @media only screen and (min-width: 480px) { /* ruleset for >= 480px */ }

The first media query in the example above will apply CSS rules when the size of the screen meets or exceeds 320 pixels. The second query will then apply CSS rules when the size of the screen meets or exceeds 480 pixels, meaning that i cna override CSS rules present in the first media query or apply additional CSS rules that are not already present in the first.

DPI - Dots Per Inch

Another media feature we can target is screen resolution. Many times, we will want to supply higher quality media, only to users with screens that can support high resolution media. Therefore to target by resolution, we can use the min-resolution, and max-resolution, media features. These media features accept a resolution value in either DPI or DPC -dots per centimeter.

An example below:
@media only screen and (min-resolution: 300dpi) { /* CSS for high resolution screens */ }

The media query in the example above targets high resolution screens by making sure the screen resolution is at least 300 DPI. if screen resolution query is met, then we can use CSS to display high resolution images and other media.

And operator

Earlier, we saw using that we can chain multiple media features of the same type in one media query by using and. We can also use and to require multiple media features. max-width of 480 and min-resolution of 300dpi can be used together. The and operator can be used to chain as many media features as necessary. The and operator can be used to chain as many media features as necessary. Each requirement has to be true before it renders the CSS within the media query.

Comma Separated List

If only one of the multiple media features in the media query has to be met, the media features can be separated in a comma separated list.

For example, if we need to apply a style, when only one of the below is true

  • The screen is more than 480 pixels wide
  • The screen is more than 480 pixels wide

The code for the requirement above can be
@media only screen and (min-width: 480px), (orientation: landscape) { /* CSS ruleset */ }

Note that the orientation media feature detects if the page has more width than height. If page is wider, consider it landscape, if page is taller, consider it portrait.

Breakpoints

The points at which media queries are set are called breakpoints. Breakpoints are the screen sizes at which your web page does not appear properly.

Rather than setting breakpoints based on specific devices, the best practice is to resize your browser to view where the website naturally breaks based on its content. The dimensions at which the layout breaks or looks odd becomes your media query breakpoint.

By observing the dimensions at which a website naturally breaks, you can set media query breakpoints that create the best possible user experience on a project by project basis, rather than forcing every project to fit a certain screen size. Different projects have different needs, and creating a responsive design should be no different. Check out this list of breakpoints by device widths on MDN. Use it as a reference of screen widths to test your website to make certain it looks great across a variety of devices.
