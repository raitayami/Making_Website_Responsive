Flexbox

What is flexbox

You have seen that, to position elements, you can use box model and CSS display properties

Flexbox, or a flexible box layout is a tool that greatly simplifies how to position elements

There are 2 important components to a flexbox layout

  1. Flex container
    • This is an element that contains flex items. All direct child elements of a flex container are flex items
    • The distinction is important as properties vary from flex item to flex containers
    • To designate an element as a flex container, the element's display property should be set to flex, or inline-flex
    • So, when an item is set to flex container, there are lots of properties you can use to specify how its children will behave.
    • The properties are
      1. Justify content
      2. Aligns-items
      3. Flex-grow
      4. Flex-shrink
      5. Flex-basis
      6. Flex
      7. Flex-wrap
      8. Align-content
      9. Flex-direction
      10. flex-flow
  2. Flex items

Flexbox shouldn't be used for the layout of an entire webpage. That is where grid comes in. People still use flexbox to layout an entire page though. It is not ideal, as you would have to nest lots of div elements, making it difficult to read

Display: flex

Any element can be a flex container. These containers are helpful tools for creating websites that respond to changes in screen sizes.

Child elements of flex containers wil change in size and location in response to the size and position of their parent container

For an element to become a flex container, its display property must be set to flex.

A div with a class that has a property of display: flex, means that div is a flex container. If they have children, the children are flex items. Also, that div will remain block level, but its child elements will not begin on new lines.

Inline-flex

A div is a block level element. Giving it the value of display: flex, it still remained a block level element

What if we wanted multiple flex containers to display inline with each other?

To make div elements not block level, we would give its display property the value of inline.

Flexbox provides the inline-flex value for the display property, allowing us to create flex containers, that are inline to each other.

Justify Content

When you change the display value of parent container to flex or inline-flex, all the child elements (flex items) moved towards the upper left corner of the parent container. This is the default behavior of flex container and their children.

Justify content helps how specify how flex item spread out form left to right. (main axis)

Values for justify content

flex-start All items positioned in order, starting from left of the parent container, with no extra space between or before them
flex-end All items positioned in order, with the last item starting from right of the parent container, with no extra space between or before them
center All items positioned in order, in the center of the parent container, no extra space between, before, or after them
space-around All items positioned in order, in the center of the parent container, no extra space between, before, or after them
space-between All items positioned in with equal space between them but no extra space before the first or after the last elements

No extra space, means that margins and borders will be respected, but no more space will be added in between elements. Other than it being specified in the style rule for the particular element.

The size of each individual flex item is not changed by this property

Align items

This property align flex items vertically within the container

The align-items property makes it possible to space flex items vertically

Here are values for the align items

flex-start All items positioned at the top of the parent container
flex-end All items positioned at the bottom of the parent container
center All items positioned at the center (halfway between the top and bottom) of the parent container
baseline The bottom of the content of all items will be aligned with each other
stretch If possible, the items will stretch from top to bottom of the container. ELements with a specified height will not stretch. Elements with a minimum height or no height specified, will stretch.

These five values will tell the elements how to behave along the cross axis of the parent container.

Cross axis stretches from top to bottom of the container.

Flex-grow

Flex items shrink proportionally when the flex container is too small

But if the container is bigger than necessary, the flex items will not stretch by default

This is where flex-grow property allows us to specify if items should grow to fill a container, and which items should grow or shrink.

I'm in the center of the flex container!

I'm on the other side of the flex container!

If max-width is set for an element, it will not grow larger than that even if there is more space for it to absorb.

All items you have seen before are declared on flex containers, or parent elements. The property. flex-grow is the first we have seen declared on flex items.

Flex-shrink

Just like flex-grow, flex shrink works the same way

When flex container is too small, flex items shrink, since the default value is 1

Flex items do not grow even if there is space since the default value for flex-grow is 0

Margins are unaffected by flex-grow, flex-shrink

Min and Max widths take precedence over flex-grow and flex-shrink

Flex-grow/shrink will onl work if the parent container is too small, or browser is adjusted

Flex-basis

So far the dimensions of the divs were determined by heights and widths from CSS

Another way to specify the width of a flex item is with the flex-basis property.

Flex-basis allows you to specify the width of an item BEFORE, it stretches or shrinks

Flex

This is shorthand property. this property allows you to declare flex-grow, flex-shrink, flex-basis in one line

If there is two values for flex, it is flex-grow and flex-shrink

Also if there is two values but the second one was a px, it is flex-grow and flex-basis

Note that there is no way to set only flex-shrink and flex-basis using 2 values

Flex-wrap

IF we do not want our content to shrink to fit the container, and we want flex items to move to the next line when necessary. This is achievable with flex-wrap property

The value are:

  • Wrap. Child elements of a flex container that don't fit in the row, will move to the next line
  • Wrap-reverse. Same functionality as wrap, but it just reverses.
  • Nowrap. Stops items from wrapping. This is the default behavior. Only used to override a wrap value.

When flex items in total is wider than the flex container, using flex-wrap as wrap, it will appear on a new line.

Align-content

Align-items property is to space flex items from top to bottom in a flex container

Align-items is for aligning elements within a single row

If a flex container has multiple rows of content, we can use align-content to space rows from top to bottom

Here are commonly used values for align-content

  • flex-start
  • flex-end
  • center
  • space-between
  • space-around
  • stretch. If no min height or height is specified, the rows of elements will stretch

Notice how the align-content property is declared on the flex containers. And how flex-wrap is used with align-content

Flex-direction

Up to this point, we have only seen flex items stretch/shrink horizontally, wrap vertically

Flex containers have two axes. A main axis and a cross axis. By default, the main axis is horizontal, and the cross axis is vertical

The main axis is used to position flex items with the following properties

  • Justify-content
  • Flex-wrap
  • Flex-grow
  • Flex-shrink

The cross axis is used to position flex items with the following properties

  • Align-items
  • Align-content

The main axis and cross axis are interchangeable

They can be switched using the flex-direction property

giving the flex-direction a property of column, the flex items will be ordered vertically.

Giving it a value of column will just make the main axis and cross axis swap, meaning, the properties such as justify content will now be on the cross axis

Flex direction can accept four values.

  • Row. Elements will be positioned from left to right
  • Row-reverse. Elements will be positioned from right to left
  • Column. Elements will be positioned from top to bottom.
  • Column-reverse. Elements will be positioned from bottom to top.

Flex direction is declared on FLEX containers

Flex-flow

Just like the flex property shorthand, flex-flow is a shorthand to declare both the flex-wrap, flex-direction properties in one line

The first value of flex-flow declaration is a flex-direction value, then a flex-wrap value.

Flex-flow property is declared on FLEX containers
